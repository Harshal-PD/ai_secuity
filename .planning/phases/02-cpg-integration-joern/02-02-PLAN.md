# Plan 02-02: Scala Taint Flow Generation & Parsing

## Objective
Write the Scala queries required by Joern to extract taint flow graphs based on static analysis locations, and parse the JSON responses back into Python dictionaries for HackerSec's `Finding` object.

## Implementation Steps

### 1. `hackersec/analysis/joern/queries.py`
- Create a function `build_taint_query(file_path: str, sink_line: int) -> str` that returns a Joern DSL query.
  - The query should identify the sink using `cpg.call.lineNumber(X)`.
  - It should map `reachableByFlows` over `cpg.identifier`.
  - The result must be serialized into a structured JSON string by the Scala runtime before it comes back to the Python HTTP response.

### 2. Update `hackersec/analysis/joern/client.py`
- Add `query(workspace: str, scala_query: str) -> dict`.
- Issue the API call to Joern's `/query` endpoint.
- Handle JSON extraction of the result, bearing in mind Joern embeds stringified JSON inside its response standard object.

### 3. Graceful Extraction
- If the sink line isn't found, return an empty array with standard `"status": "no_flow_found"` to prevent pipeline halts.

## Verification
- Add unit test logic wrapping the query generator string logic and ensuring it's syntactically valid Scala DSL string output.
