# Plan 02-03: Celery Pipeline Integration & Database Enrichment

## Objective
Update Celery task execution to query Joern for CPG traces of each deduplicated Finding, then persist them to SQLite.

## Implementation Steps

### 1. `hackersec/worker/tasks.py`
- Immediately after `dedup_findings(raw_findings)`, initialize the `JoernClient`.
- Wrap Joern calls in a blanket `try/except` to gracefully bypass parsing failures or network timeouts via the exceptions defined in 02-01.
- If Joern is down or failed, add `{"cpg_status": "failed", "reason": "..."}` to the finding's `cpg_context`.
- Iterate through each finding. Use `client.query_taint()` via the finding's `file_path` and `line_start`.
- Assign the result dict directly to `finding.cpg_context`.

### 2. `hackersec/db/store.py`
- Modify `save_findings` and `get_findings` to serialize/deserialize `cpg_context` using `json.dumps()` and `json.loads()`.

## Verification
- Unit test ensuring `JSONDecodeError` won't break the database schema when loading older findings without context.
