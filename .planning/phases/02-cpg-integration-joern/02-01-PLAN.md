# Plan 02-01: Joern Server Service & Client Wrapper

## Objective
Integrate Joern HTTP server mechanics into HackerSec and build the foundational Python wrapper via `httpx`. Since Joern takes too long to cold start, we use its HTTP REST server.

## Context
- Joern provides a REST interface when launched with `joern --server`.
- The Python client will post source code paths to create workspaces and send arbitrary Scala strings to execute.

## Implementation Steps

### 1. `hackersec/analysis/joern/client.py`
- Create `JoernClient` class initializing with `base_url` (default: `http://localhost:9000`).
- Use `httpx.Client()` for synchronous requests.
- Implement `ping()` method hitting an info/health endpoint.
- Implement `create_workspace(name: str)` hitting the workspace creation API.
- Implement `import_code(workspace: str, target_path: str)` hitting the code import API.

### 2. `hackersec/analysis/joern/exceptions.py`
- Create `JoernConnectionError` and `JoernQueryError` classes for graceful fallback logic later.

### 3. Updates to `requirements.txt`
- Ensure `httpx>=0.27.0` is listed (it is already there from Phase 1, but confirm).

## Verification
- Initialize `JoernClient()` in a local Python shell and assert `ping()` returns False when port 9000 is down.
