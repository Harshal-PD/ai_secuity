# Plan 03-03: Celery Pipeline Integration

## Objective
Embed the finalized FAISS querying mechanisms natively inside our analysis orchestrator to augment the findings with high fidelity RAG descriptions.

## Implementation Steps

### 1. `hackersec/worker/tasks.py`
- Expose global instantiation of `LocalRAGStore` outside the celery task loops to avoid cold booting matrix encodings on every upload. Let python workers maintain the memory arrays statitically on boot.
- Underneath Step 3.5 (CPG Enrichement) map Step 3.7: RAG Enrichment.
- Loop backwards over the deduped findings:
   - Compile query strings bridging Semgrep extracted CWE IDs and categorized OWASP strings: e.g., `Explaining vulnerabilities of {f.cwe_ids} and securing it.`
   - Bind the returned metadata context chunks directly onto `finding.rag_docs`.

### 2. `hackersec/db/store.py`
- Revalidate that `json.dumps()` securely captures textual strings formatted through `LocalRAGStore.search()`.

## Verification
- Monitor memory consumption making sure celery workers don't overflow mapping 384 dimensions redundantly across loops.
