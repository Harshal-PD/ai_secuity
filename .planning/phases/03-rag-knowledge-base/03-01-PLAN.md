# Plan 03-01: FAISS Vector Store Index Service

## Objective
Implement `LocalRAGStore` within `hackersec/analysis/rag.py` capable of loading `all-MiniLM-L6-v2`, embedding queries natively via `sentence-transformers`, tracking metadata matrices, and wrapping the C++ `FAISS` library abstractions.

## Implementation Steps

### 1. `hackersec/analysis/rag.py`
- Initialize `SentenceTransformer('all-MiniLM-L6-v2')`.
- Expose an `add_documents(documents: list[dict])` method where dict objects enforce a `{"text": "...", "id": "CWE-..."}` schema.
- Map the embedded vectors directly into `faiss.IndexFlatL2(384)`.
- Serialize metadata arrays alongside FAISS `.bin` outputs inside a `data/` volume block to preserve linkage on instantiation.
- Implement `search(query: str, top_k=2)` that:
  1. Computes the embedding vector of the search query.
  2. Runs `index.search(embeddings, top_k)`.
  3. Maps FAISS position integers back against the serialized metadata arrays to return raw context tuples.

## Verification
- Validate the embedding library installs cleanly and performs FAISS lookups mapping arrays accurately without Langchain bloating logic.
