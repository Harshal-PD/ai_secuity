# Plan 03-02: Knowledge Base Seed Scripts

## Objective
Generate localized scripts building baseline security data context dicts mirroring CWE taxonomy, allowing pipeline bootstrap on any machine natively.

## Implementation Steps

### 1. `hackersec/ingestion/seed_kb.py`
- Draft an independent script capable of populating our baseline RAG database.
- Define a dictionary hard-coding the OWASP Top 10 summaries (e.g. Broken Access Control, SQL Injection) and Top 25 CWE summaries (e.g., CWE-89, CWE-79, CWE-22, CWE-78).
- Invoke `LocalRAGStore(index_path="data/kb.faiss")`.
- Call `store.add_documents()` iterating over the generated dataset context.
- Implement an initialization hook binding the script under Makefile (`make seed-kb`).

## Verification
- Run `python hackersec/ingestion/seed_kb.py` and manually check that `data/kb.faiss` and `data/kb_meta.json` binaries spawn.
