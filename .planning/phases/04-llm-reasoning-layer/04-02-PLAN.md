# Plan 04-02: LLM Output JSON Parser

## Objective
Implement `hackersec/analysis/llm/parser.py` focusing on safely extracting and validating JSON structures outputted by DeepSeek-Coder bounding unexpected markdown artifacts and casting variables strictly.

## Implementation Steps

### 1. `LLMResult` Schema Validation
- Establish a python `dataclass` or custom dict mapping:
  - `explanation: str`
  - `root_cause: str`
  - `fix_suggestion: str`
  - `confidence: float` (range `0.0` to `1.0`)
- Write parser logic `parse_llm_response(raw_text: str) -> dict`.

### 2. Error Recovery Mappings
- `deepseek` frequently answers in ```json ... ``` blocks instead of raw outputs.
- Build regex extraction (`r'```json(.*?)```'`) catching blocks if direct `json.loads` natively throws decoding exceptions.
- Provide a default gracefully degraded dict: `{"llm_status": "failed_parsing", "explanation": "Unable to decode AI payload"}` if bounds are shattered.

## Verification
- Can be tested via feeding fake string outputs (`"Here is the result:\n```json\n{\"confidence\": 0.9}\n```"`) to evaluate robustness.
