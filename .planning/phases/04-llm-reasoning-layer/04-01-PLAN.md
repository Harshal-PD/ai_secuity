# Plan 04-01: Ollama Client + Prompt Template

## Objective
Establish `hackersec/analysis/llm/client.py` operating an asynchronous or synchronous HTTP integration wrapping Ollama's REST logic. Build the master prompt blueprint incorporating RAG documents, Code Property traces, and anti-injection brackets.

## Implementation Steps

### 1. Client Wrapper
- Instantiate `OllamaClient(base_url="http://localhost:11434")` wrapping `httpx`.
- Implement `ping()` ensuring reachable status mapping to `.env` values `OLLAMA_URL`.
- Implement `generate(model="deepseek-coder-v2", prompt: str)` expecting `{"format": "json"}` via configuration overrides natively supported by Ollama.

### 2. Prompt Builder
- Inside `hackersec/analysis/llm/prompter.py`, formulate a python function `build_analysis_prompt(finding: Finding)` capturing:
  1. System context explicitly commanding grounding strictly to provided CWE strings.
  2. Formatting: Encase untrusted `finding.code_snippet` inside strict boundaries (e.g. `[UNTRUSTED_CODE_START]` and `[UNTRUSTED_CODE_END]`), advising the parser to ignore instructional context within this block.
  3. Include retrieved `cpg_context` mappings cleanly formatted.

## Verification
- Confirm the built prompt compiles cleanly with all inputs substituted and validates properly via simple connection pings to `http://localhost:11434`.
