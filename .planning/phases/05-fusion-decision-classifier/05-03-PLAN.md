# Plan 05-03: Inference Service & SHAP

## Objective
Embed `joblib` loading architectures directly inside `hackersec/analysis/ml/inference.py` ensuring singleton caching limits RAM consumption, formatting the exact model bounds around Celery Pipeline bindings alongside `SHAP`.

## Implementation Steps

### 1. `FusionClassifier` Class
- Build `FusionClassifier` encapsulating `joblib.load("data/fusion_model.joblib")`.
- Expose `predict(finding: Finding)` triggering `extract_features(finding)`.
- If prediction probability outputs `>= 0.5` (`"true_positive"`), initialize `shap.TreeExplainer(self.model)`.
- Compute explicitly isolating `shap_values(features)` outputting `shap_dict = {"static": val, "llm": val, "cpg": val}` to `finding.fusion_verdict`.

### 2. Celery Pipeline
- Insert `Step 3.9: ML Fusion` in `hackersec/worker/tasks.py`.
- Instantiate `FusionClassifier()` outside the finding iteration bounds.
- Feed mappings executing `predict(f)`. Assign resulting predictions mapping string metrics onto arrays. Update `f.fusion_verdict` strictly.
